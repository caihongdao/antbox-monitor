# 矿机冷却系统监控平台 - 仪表盘原型

基于UI设计规范创建的交互式仪表盘原型，用于展示监控系统的核心功能。

## 原型功能

### 已实现功能
1. **总览仪表盘** - 基于设计规范中的总览页面布局
2. **实时数据展示** - 模拟150个矿机站点的监控数据
3. **关键指标卡片** - 6个核心KPI卡片展示
4. **趋势图表** - 4个ECharts图表展示24小时趋势
5. **告警管理** - 交互式告警列表和操作
6. **站点状态分布** - 饼图展示站点健康状况
7. **响应式设计** - 适配不同屏幕尺寸

### 核心组件
- **顶部导航栏**: 系统标识、通知中心、用户信息、实时时间
- **KPI卡片区**: 总站点数、在线站点、告警数量、总算力、总功耗、平均温度
- **趋势图表区**: 功耗、算力、温度、在线站点变化趋势
- **告警列表**: 最新告警事件及处理操作
- **状态分布**: 站点状态统计和分布图

## 技术实现

### 前端技术栈
- **HTML5/CSS3**: 语义化标签和现代CSS特性
- **纯JavaScript**: 原生JS实现交互逻辑
- **ECharts 5**: 数据可视化图表库
- **Font Awesome**: 图标库
- **CSS变量**: 基于设计规范的颜色、间距、字体系统

### 设计系统实现
- **色彩方案**: 完整实现工业蓝主题
- **间距系统**: 8px基准的间距阶梯
- **字体系统**: 多字体fallback和字号阶梯
- **阴影系统**: 5级阴影定义
- **响应式**: 支持大屏幕、中等屏幕、小屏幕适配

### 交互特性
- **实时时间更新**: 每秒更新系统时间
- **数据采集倒计时**: 模拟60秒数据采集周期
- **图表响应式**: 窗口大小变化时自动调整
- **告警操作**: 处理、忽略、查看等交互功能
- **悬停效果**: 卡片悬停动画和状态反馈

## 快速启动

### 方法1：直接打开HTML文件
```bash
# 在Chrome、Firefox等现代浏览器中直接打开
open dashboard.html  # macOS
xdg-open dashboard.html  # Linux
start dashboard.html  # Windows
```

### 方法2：使用Python HTTP服务器
```bash
# Python 3
python3 -m http.server 8000

# 或者指定端口
python3 -m http.server 8080

# 然后在浏览器中访问
# http://localhost:8000/dashboard.html
```

### 方法3：使用Node.js HTTP服务器
```bash
# 全局安装http-server
npm install -g http-server

# 启动服务器
http-server -p 8080
```

## API集成准备

### 数据端点设计
```javascript
// 仪表盘总览数据
GET /api/dashboard/overview
Response: {
  total_sites: 150,
  online_sites: 148,
  active_alerts: 3,
  total_hashrate: "7.24 PH/s",
  total_power: "152.8 MW",
  avg_temperature: "34.7°C"
}

// 趋势数据
GET /api/dashboard/trend?metric=power&hours=24
GET /api/dashboard/trend?metric=hashrate&hours=24
GET /api/dashboard/trend?metric=temperature&hours=24

// 告警列表
GET /api/alerts?status=active&limit=10

// 站点状态分布
GET /api/sites/status/distribution
```

### 后端适配器
原型已设计为可通过以下方式轻松集成真实数据：

1. **替换模拟数据函数**: 将`generateRandomData()`替换为API调用
2. **添加数据加载器**: 实现`fetchDashboardData()`函数
3. **配置API端点**: 在配置文件或环境变量中设置后端URL
4. **错误处理**: 添加加载状态和错误重试机制
